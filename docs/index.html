The Problem

The data exists.
The interface doesn't.

GBIF holds 2 billion biodiversity records. OBIS holds 100 million marine species records. NOAA maintains decades of stranding data. The gap is not access — it's presentation.

01

Fragmented databases

Sighting records from research institutions, stranding reports from coastal networks, acoustic detections from hydrophone buoys — scattered across dozens of systems with no unified view.

02

No accessible interface

Exploring open cetacean data requires writing API queries, downloading CSV files, and loading them into GIS software. Conservation advocates can't do this. They need a map.

03

Missing conservation context

Raw sighting coordinates are not enough. Conservationists need to see where feeding grounds overlap with shipping lanes, where sonar zones correlate with stranding clusters.

04

No historical baseline

19th century whaling logs contain distribution data that predates industrial hunting by a century. Nobody has made this comparable to modern records in a single visual.

The Platform

Every feature
had a reason.

Nothing was added for visual effect. Every feature solves a real problem in how open ocean data gets communicated to non-technical audiences.

🌍

3D Globe projection

Migration routes on a flat map are lines. On a globe they become arcs curving across the surface of the Earth — the scale of a 5,000km whale journey becomes viscerally real.

📍

25,000 GPU-rendered dots

All records render as a single GPU layer — not 25,000 DOM elements. Smooth at 60fps on any device. WebGL via @maptiler/sdk on MapLibre GL JS v5.

🌿

Conservation overlay layers

Toggle feeding grounds, sonar exercise zones, and 32 shipping corridors onto the same globe as the sighting data. The overlaps speak for themselves.

📅

Historical to present filter

Slide from 1785 to 2026. Watch the distribution of Blue whale records shift as you move from 19th century whaling grounds to modern sighting surveys.

🔔

Weekly email alerts

Subscribe to new sighting digests filtered by species, region, or data layer. Sent via AWS SES with double opt-in, per-subscriber preferences, one-click unsubscribe.

⚡

Automated data pipeline

Six sync scripts run on cron schedules: GBIF weekly, OBIS weekly, strandings monthly, acoustics monthly, iNaturalist weekly, historical yearly. Zero manual intervention.
