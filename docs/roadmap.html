Overview

Where the platform is going

This roadmap outlines how whaledata.org will evolve — from expanded whale datasets and improved geospatial tools to future API capabilities, acoustic layers, and research integrations.

The focus is on building a platform that stays open, useful, and scalable for developers, researchers, and conservation work. Explore the current data sources and technology stack behind what is already live.

Near term

Features actively in development or ready to begin in the next sprint cycle.

1

API usage logging + admin panel tab

Per-endpoint request logging with timestamp, endpoint, response time, and status code. Admin panel tab showing usage trends over time. Currently partially implemented — the middleware was rewritten to avoid a Starlette incompatibility but the database table needs manual creation.

In progress
2

NOAA PACM direct integration

The Passive Acoustic Cetacean Monitoring API was blocked by the hosting network at deployment. Resolving this would expand the acoustic layer from ~750 records to potentially 15,000+, dramatically improving coverage for deep-diving species like Sperm whale.

Blocked
3

Embed widget

A filtered globe view embeddable on third-party conservation and research websites via a simple iframe or JavaScript snippet. Users would be able to configure species, region, and layer filters via URL parameters. Targeted at NGOs who want to embed relevant data on their own sites.

Planned
4

Redis query cache

Caching the full sightings response for 5 minutes would reduce database load by ~80% under typical usage and bring average response time from 6.8s to under 1s for cache hits. The Docker-compose setup is already containerised — adding a Redis service is a one-session change.

Planned

Medium term

Features planned for the next 3–6 months.

5

Dynamic habitat predictions

Integrating sea surface temperature and chlorophyll concentration data from NOAA/Copernicus to generate predicted whale habitat zones that update weekly. This would move the feeding ground layer from static polygons to data-driven predictions — a significant conservation research upgrade.

Planned
6

Additional species coverage

Expanding beyond the current 6 species to include Minke whale, Bowhead whale, Narwhal, and Beluga — all with significant IUCN conservation concern. The data pipeline infrastructure already supports adding new species via GBIF taxon key configuration.

Planned
7

Individual whale tracking integration

Aggregating published satellite tag tracks from research programmes (Happy Whale, SPLASH, Pacific Whale Foundation) to show individual whale movement paths alongside population-level sighting data. Requires data sharing agreements with participating research institutions.

Planned
8

Sighting submission form

Allowing verified researchers and citizen scientists to submit sightings directly to whaledata.org, with a review queue in the admin panel. Would create a direct data contribution pathway rather than relying entirely on GBIF/OBIS synchronisation schedules.

Planned

Public API access

All data in whaledata.org is accessible via the public REST API at api.whaledata.org. No authentication is required. All endpoints return JSON.

Available endpoints

The full OpenAPI documentation is available at api.whaledata.org/docs

GET/sightings/All sightings with optional species, date, limit filters
GET/speciesAll six species with IUCN status and descriptions
GET/species/{id}Single species detail
GET/routes/Migration routes as GeoJSON LineStrings
GET/strandings/NOAA stranding events
GET/acoustics/Acoustic detection records
GET/layers/summaryRecord counts across all five data layers
POST/alerts/subscribeSubscribe to weekly sighting digest emails

How to contribute

whaledata.org is open source. The codebase, data pipeline scripts, and database schema are all on GitHub. Contributions — whether that's a data quality fix, a new sync script for an additional source, or an improvement to the frontend — are welcome.

⚙️

GitHub repository

The full codebase — API, frontend, sync jobs, Docker config, and documentation source files. Open issues for bugs or feature requests.
📡

API documentation

Full OpenAPI / Swagger docs for the public REST API. Build your own tools on top of the same 25,000+ record dataset.
🌍

Live platform

The platform itself — explore the data, test the conservation layers, subscribe to alerts. Bug reports welcome via the contact form.
📧

Get in touch

Researchers, conservation organisations, or developers who want to discuss data partnerships, integrations, or contributions — reach out directly.

The ocean is not well served by data that sits in archives.

Every one of the 25,000+ records in whaledata.org was a real observation — a researcher on a survey vessel, a citizen scientist on a whale watch boat, a hydrophone listening in deep water, a stranding coordinator on a beach at dawn. Behind each point on the globe is a person who noticed something and wrote it down.

The purpose of this platform is to make those observations visible — to connect the global pattern of where whales have been seen with the human activities that threaten them, and to do it in a way that anyone can engage with. Not just researchers. Not just policy makers. Anyone who cares about the ocean.

If you've read this far, you now know more about how this was built than most people who have ever visited whaledata.org. That knowledge belongs to you. Use it to build something, share something, or simply to understand the ocean a little better.
